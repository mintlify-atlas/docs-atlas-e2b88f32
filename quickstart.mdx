Get up and running with FSM by building a simple counter app that demonstrates reactive state management with stream builders and listener mixins.

## What you'll build

A Flutter counter app that uses FSM to manage state reactively, demonstrating:
- Document-based state storage
- Real-time UI updates with stream builders
- Automatic subscription management with listener mixins

## Prerequisites

Before you begin, make sure you have:
- Flutter 3.0.0 or later installed
- FSM package added to your project (see [Installation](/installation))

## Complete example

Here's a complete working counter app using FSM. Copy this into your `lib/main.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:fsm/fsm.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'FSM Counter Demo',
      theme: ThemeData(
        colorScheme: ColorScheme.fromSeed(seedColor: Colors.deepPurple),
        useMaterial3: true,
      ),
      home: const MyHomePage(title: 'FSM Counter'),
    );
  }
}

class MyHomePage extends StatefulWidget {
  const MyHomePage({super.key, required this.title});

  final String title;

  @override
  State<MyHomePage> createState() => _MyHomePageState();
}

class _MyHomePageState extends State<MyHomePage> with FListenerStateMixin {
  @override
  void initState() {
    super.initState();
    // Listen to counter changes with automatic cleanup
    listenF(F.instance.collection('home').doc('counter'), (data) {
      DocumentSnapshotWrapper snapshotWrapper = data;
      debugPrint('Counter updated: ${snapshotWrapper.data}');
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        backgroundColor: Theme.of(context).colorScheme.inversePrimary,
        title: Text(widget.title),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            const Text(
              'You have pushed the button this many times:',
            ),
            // FStreamBuilder automatically rebuilds on data changes
            FStreamBuilder<int>(
              stream: F.instance.collection('home').doc('counter'),
              initialData: 0,
              builder: (context, value) {
                return Text(
                  value.toString(),
                  style: Theme.of(context).textTheme.headlineMedium,
                );
              },
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          // Get current value and increment
          final currentValue = 
              F.instance.collection('home').doc('counter').snapshot.data ?? 0;
          F.instance.collection('home').doc('counter').setData(currentValue + 1);
        },
        tooltip: 'Increment',
        child: const Icon(Icons.add),
      ),
    );
  }
}
```

## How it works

Let's break down the key parts of this example:

<Steps>
  <Step title="Import FSM">
    ```dart
    import 'package:fsm/fsm.dart';
    ```
    Import the FSM package to access all state management features.
  </Step>

  <Step title="Add the listener mixin">
    ```dart
    class _MyHomePageState extends State<MyHomePage> with FListenerStateMixin {
    ```
    The `FListenerStateMixin` provides automatic subscription management. All subscriptions created with `listenF()` are automatically cancelled when the widget is disposed.
  </Step>

  <Step title="Listen to state changes">
    ```dart
    listenF(F.instance.collection('home').doc('counter'), (data) {
      DocumentSnapshotWrapper snapshotWrapper = data;
      debugPrint('Counter updated: ${snapshotWrapper.data}');
    });
    ```
    Set up a listener in `initState()` to log whenever the counter changes. This subscription is automatically cleaned up when the widget is disposed.
  </Step>

  <Step title="Build reactive UI with FStreamBuilder">
    ```dart
    FStreamBuilder<int>(
      stream: F.instance.collection('home').doc('counter'),
      initialData: 0,
      builder: (context, value) {
        return Text(value.toString());
      },
    )
    ```
    `FStreamBuilder` automatically rebuilds the widget whenever the counter document changes. The `initialData` provides a default value before any data is set.
  </Step>

  <Step title="Update state">
    ```dart
    final currentValue = 
        F.instance.collection('home').doc('counter').snapshot.data ?? 0;
    F.instance.collection('home').doc('counter').setData(currentValue + 1);
    ```
    Update the counter by getting the current value and setting a new value. This triggers all listeners and stream builders.
  </Step>

  <Step title="Run the app">
    ```bash
    flutter run
    ```
    Run your app and tap the floating action button to see the counter increment reactively!
  </Step>
</Steps>

## Key concepts demonstrated

### Firebase-inspired API

FSM uses a familiar path-based API inspired by Firebase:

```dart
F.instance.collection('home').doc('counter')
```

This creates a reference to a document at path `home/counter`.

### Reactive updates

When you call `setData()`, all listeners and stream builders are automatically notified:

```dart
// This update triggers FStreamBuilder to rebuild
F.instance.collection('home').doc('counter').setData(42);
```

### Automatic cleanup

The `FListenerStateMixin` automatically cancels subscriptions when the widget is disposed, preventing memory leaks:

```dart
class _MyHomePageState extends State<MyHomePage> with FListenerStateMixin {
  // All listenF() subscriptions are automatically cancelled on dispose
}
```

## Try these experiments

Now that you have a working app, try these modifications:

<Accordion title="Add a reset button">
  Add a button that resets the counter to zero:

  ```dart
  ElevatedButton(
    onPressed: () => F.instance.collection('home').doc('counter').setData(0),
    child: const Text('Reset'),
  )
  ```
</Accordion>

<Accordion title="Use FDocumentStreamBuilder">
  Replace `FStreamBuilder` with `FDocumentStreamBuilder` for a more concise syntax:

  ```dart
  FDocumentStreamBuilder<int>(
    document: 'home/counter',
    initialData: 0,
    builder: (context, value) {
      return Text(value.toString());
    },
  )
  ```
</Accordion>

<Accordion title="Add multiple counters">
  Create separate counters for different purposes:

  ```dart
  FDocumentStreamBuilder<int>(
    document: 'home/taps',
    initialData: 0,
    builder: (context, taps) => Text('Taps: $taps'),
  )
  
  FDocumentStreamBuilder<int>(
    document: 'home/swipes',
    initialData: 0,
    builder: (context, swipes) => Text('Swipes: $swipes'),
  )
  ```
</Accordion>

<Accordion title="Store complex data">
  Store a Map with multiple fields:

  ```dart
  F.instance.collection('home').doc('stats').setData({
    'count': 0,
    'lastUpdated': DateTime.now().toIso8601String(),
    'user': 'Guest',
  });
  ```
</Accordion>

## Troubleshooting

<Accordion title="Counter doesn't update when I tap the button">
  Make sure you're calling `setData()` with a different value. FSM skips updates when the data hasn't changed:

  ```dart
  // ✅ Good - increments the value
  final current = doc.snapshot.data ?? 0;
  doc.setData(current + 1);

  // ❌ Bad - sets the same value repeatedly
  doc.setData(0);
  ```
</Accordion>

<Accordion title="Memory leaks or subscriptions not cleaning up">
  Ensure you're using the `FListenerStateMixin` or manually calling `closeF()`:

  ```dart
  // ✅ Good - automatic cleanup
  class _MyState extends State<MyWidget> with FListenerStateMixin {
    // listenF() subscriptions are auto-cancelled
  }

  // ❌ Bad - no cleanup
  class _MyState extends State<MyWidget> {
    // Subscriptions will leak!
  }
  ```
</Accordion>

<Accordion title="FStreamBuilder shows initial value even after update">
  Check that your `builder` function is using the `value` parameter, not accessing the document directly:

  ```dart
  // ✅ Good - uses the value parameter
  FStreamBuilder<int>(
    stream: F.instance.doc('path'),
    builder: (context, value) => Text('$value'),
  )

  // ❌ Bad - ignores the value parameter
  FStreamBuilder<int>(
    stream: F.instance.doc('path'),
    builder: (context, value) => Text('${F.instance.doc('path').snapshot.data}'),
  )
  ```
</Accordion>
